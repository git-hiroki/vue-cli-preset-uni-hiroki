# 安全性
Hiroki 非常重视我们软件产品和服务的安全性, 包括通过GitHub组织管理的所有代码库等。

# 潜在安全检查点
1. 不上传密钥文件至 git 仓库
2. 删除远端分支时, 请再三确认
3. 如无必要, 不使用强制推送, 如需强制推送, 请确保向至少一位 leader 进行报备
4. 引入第三方开源代码前确认开源协议, 做好风控, 规避潜在的法律风险
5. 不对应私有保管的技术资产开放公开权限, 包括但不仅限于 git 账号, 微信开发者账号等
6. 注意使用 sitemap.json 限制未登录不应允许访问的页面, 以避免被爬虫收录带来安全问题
